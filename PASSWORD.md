# PASSWORD
## What is Password?
A password is a secret word, phrase, or combination of characters that a person uses to prove their identity and gain access to something.
It can be used to:

Unlock a phone

Log into an email or social media account

Access a computer

Open a secure file

## When you log into:

Email

Banking apps

Social media

Servers

WiFi

Admin panels

You are using a password as an authentication mechanism.

## Role of Passwords in Cybersecurity

Passwords support the CIA Triad:

Confidentiality – Prevents unauthorized access to data

Integrity – Protects data from being modified

Availability – Ensures access only to legitimate users

Without strong passwords:

Systems can be hacked

Data can be leaked

Identity theft can occur

Financial fraud becomes easy

## Types of Passwords
1. Text-Based Passwords

Most common type
Example: Cyber@123

2. PIN (Personal Identification Number)

Numeric only
Example: ATM PIN

Used in banking systems, mobile locks.

3. Passphrase

A long sentence instead of a short password.

Example:
I_Love_Cybersecurity_2026!

Much stronger than:
Cyber123

4.One-Time Password (OTP)

Temporary password valid for one session or short time.

Used in:

Banking apps

Email verification

2FA systems

Example: 6-digit SMS code.

5. Graphical Passwords

User selects images or patterns.

Used in:

Android pattern lock

Some secure login systems

## How Passwords Are Stored Securely

-Important: Websites do NOT store passwords in plain text (if secure).

They use:

1.Hashing

Hashing converts password into a fixed-length string using algorithms like:

bcrypt

SHA-256

Argon2

Example:

Password:

Cyber@123

Hashed:

9f86d081884c7d659a2feaa0c55ad015...

2.Salting

Salt = Random data added before hashing.

Why?
To prevent:

Rainbow table attacks

Precomputed hash attacks

## Characteristics of a Strong Password

A strong password should:

✔ Be at least 12–16 characters long
✔ Include uppercase and lowercase letters
✔ Include numbers
✔ Include symbols
✔ Not contain personal information
✔ Not reuse previous passwords

## Password Policies in Organizations

Companies enforce policies like:

Minimum length requirement

Complexity requirement

Expiration period

Account lockout after 3–5 failed attempts

Multi-Factor Authentication (MFA)

## Multi-Factor Authentication (MFA)

Passwords alone are not enough today.

MFA combines:

Something you know → Password

Something you have → OTP / phone

Something you are → Biometrics

## Risks of Weak Passwords

Weak passwords can lead to:

Identity theft

Data breaches

Financial fraud

Ransomware attacks

Server compromise

Reputation damage

## Future of Passwords

Passwords are slowly being replaced by:

Biometrics

Hardware tokens

Passkeys (FIDO-based authentication)

Zero-trust models.

## Conclusion:

Passwords are an essential part of digital life, used to protect personal information and restrict access to devices, accounts, and systems. 
They serve as a simple yet powerful method of verifying identity. 
However, the effectiveness of a password depends on how strong and securely it is created and maintained. 
Using unique, complex passwords and keeping them confidential helps ensure better protection and privacy.
