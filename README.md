## Objective

Learn how password complexity affects security by:

Creating multiple passwords

Testing them using strength checkers

Researching common attacks

Documenting everything in GitHub.

## TOOLS USED

## STEPS FOLLOWED
Create Test Passwords-8 Passwords with increasing complexity.
Test Password Strength using Kaspersky,NordPass and Security.org
Record Scores,Estimated Crack Time and Feedback given by tool.

## OBSERVATIONS

- Short passwords are cracked quickly.
- Adding uppercase letters increases complexity.
- Symbols significantly increase crack time.
- Length is more important than complexity alone.
- Random passwords are strongest.

##  Common Password Attacks

### 1️.Brute Force Attack
Attacker tries every possible combination.
Longer passwords take exponentially more time.

### 2️.Dictionary Attack
Uses common words and leaked password lists.
Simple words are easily cracked.

### 3️.Credential Stuffing
Using leaked passwords on multiple sites.

## Best Practices Learned

- Use at least 12–16 characters
- Combine uppercase, lowercase, numbers, symbols
- Avoid common words
- Avoid personal information
- Use passphrases
- Enable Multi-Factor Authentication (MFA)
- Use a password manager

## Conclusion

Password complexity directly affects security. 
Long and random passwords significantly increase resistance against brute force and dictionary attacks.
